# Concepts

> *Ideas that live at the edge of what the Spectorium is becoming.*

The `concepts/` folder holds foundational idea documents — the thinking behind the platform's larger systems. These are not standards (which govern how things are done) and not lore (which governs how things are named). Concepts govern *why* the deeper architecture is shaped the way it is.

## What Belongs Here

A concept document belongs in `concepts/` when:

- It explains a structural principle that spans multiple parts of the platform
- It references an external system or repository
- It is foundational to future development but not yet implemented
- It is too large an idea for a single standard or Grove file

---

## Adding a Concept

1. Create `concepts/<CONCEPT_NAME>.md`
2. Add it to the index table above
3. Reference it from any relevant platform file
4. Link to it from `GLOSSARY.md` if it introduces new terms
